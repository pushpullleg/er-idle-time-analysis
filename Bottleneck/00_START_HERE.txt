## ‚ö†Ô∏è CRITICAL: Doctor "Idleness" is CONDITIONAL, Not Simple

**READ THIS FIRST:** `/Bottleneck/DOCTOR_IDLE_DEFINITION.md`

**Key Point:**
Do NOT assume: "Doctor finished + 10-min buffer + Patient waiting = Doctor Idle"

**INSTEAD check ALL simultaneously:**
- Doctor has capacity? (idle_doctors > 0)
- Patient is waiting? (waiting_patients > 0)
- Bed IS available? (available_beds > 0)

**Only if ALL are TRUE:** ‚Üí Coordination failure (process fixable)

**If ANY fails:** ‚Üí Resource constraint (needs hiring/expansion)

**Why this matters:**
- Prevents false conclusions
- Distinguishes between fixable process vs. capital-needs
- Gives management confidence in analysis

See DOCTOR_IDLE_DEFINITION.md for full walkthrough with examples.

## The Approach We've Documented

### **Understanding Randomness**

We identified that your ER model depends on **4 factors:**
```
Wait = f(Severity, Test Turnaround, Room Availability, Staff Workload)
```

Most analysis tools assume ONE bottleneck. We can't.

**Industry Solutions:**
1. **Deterministic (Ours - Phase 1-4):** Define rules, acknowledge limitations
2. **Process Mining (If needed):** Track actual flow, find real bottleneck
3. **Queuing Simulation (If needed):** Model ER mathematically
4. **Machine Learning (If needed):** Learn patterns from data

### **Our Bed Occupancy Algorithm**

**Key innovation:** We'll calculate when beds are occupied (patient Doctor_Seen ‚Üí Exit) and when available.

**Logic:**
```
Bottleneck instance =
  Patient waiting (Triage End ‚Üí Doctor Seen)
  AND Available doctor (finished + 10-min buffer)
  AND Available bed (occupancy < capacity)
  ‚Üí This is a PROCESS failure, not resource failure
```

### **Handling Unknowns**

We can't measure:
- ‚úó Test result timing (is patient waiting for lab?)
- ‚úó Room assignment (which patient in which room?)
- ‚úó Doctor specialization (can any doctor see any patient?)

**Solution:** Document these as assumptions, flag instances we can't explain, propose advanced methods if needed.
